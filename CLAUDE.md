# כללי עבודה בריפו הזה

## כלל ברזל — פריסה מיידית
בכל פעם שמבקשים ממני לבצע שינוי בקוד (בעיקר ב-`index.html`), אחרי שהשינוי מוכן:

1. **למזג ל-`main`** (ולדחוף/push ל-`origin/main`).
2. **מיד אחר כך לעדכן את `sw.js`** — להעלות את מספר הגרסה במשתנה `CACHE` (למשל `gan-receipts-v124` → `gan-receipts-v125`), ולמזג/לדחוף גם את זה ל-`main`.

המטרה: מכיוון שה-Service Worker משתמש ב-network-first לדף הראשי אבל cache-first לשאר הנכסים, שינוי בגרסת ה-CACHE הוא מה שמבטיח שהמשתמש יראה את הגרסה החדשה של האפליקציה מיד (ולא גרסה ישנה מה-cache). בלי לעדכן את מספר הגרסה, ייתכן שהשינוי לא יורגש בפועל באפליקציה המותקנת.

זהו כלל קבוע — יש לפעול לפיו בכל בקשת עדכון, בלי לשאול שוב בכל פעם.
